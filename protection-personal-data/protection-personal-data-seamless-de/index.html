Datenschutzinfo für Checkout Seamless

Datenschutzinfo für Checkout Seamless

Auf dieser Infoseite können Sie sich über alle Daten informieren, die das Produkt Checkout Seamlessverarbeitet.

Bei der Bestellung des Produkts Checkout Seamless kann unterschieden werden zwischen

  • 1 Basisprodukt, optional erweitert um

  • 1 bis N Zahlungsmitteln, und/oder

  • 0 bis N optionalen Zusatzfeatures

Sollten Sie darüber hinaus noch Fragen zu datenschutzrechtlichen Details der einzelnen Parameter haben (zB betroffene Person, Datenkategorie etc), können Sie diesbezügliche Informationen unter individual parameter einsehen.

Basisprodukt

Datenverarbeitung je Zahlungsmittel

Unter Auswahl eines Zahlungsmittels können Sie hier simulieren, welche Daten (Parameter) für eine Zahlungstransaktion mit einem bestimmten Zahlungsmittel verarbeitet werden. Angezeigt werden dabei sowohl Daten,

  • die vom Merchant oder dem Endkunden an die hobex Service IT Zahlungsschnittstelle gesendet werden (sog. Input-Daten)

  • die von hobex Service IT wieder an diese zurück gehen (sog. Output-Daten).

Sämtliche hier angegebenen Parameter müssen verarbeitet werden, um das Produkt Checkout Seamless in seiner Basisversion dem Merchant zur Verfügung stellen zu können (Zweck der Datenverarbeitung).

Die hier angezeigten Parameter unterteilen sich in Parameter

  • die unabhängig vom Zahlungsmittel verarbeitet werden (Standard)

  • die abhängig vom jeweiligen Zahlungsmittel verarbeitet werden (Zahlungsmittel)

  • die vom Endkunden eingegeben werden (Interaktion Endkunde)

Inklusivleistungen

In der Basisfunktion von Checkout Seamless inkludiert sind bereits einige Funktionen, die nachfolgend näher dargestellt werden.

Table 28. Inklusivleistungen
Inklusivleistung Verarbeitete Daten und Datenschutzinfos

1. Rückgabe Karteninhaber Kreditkarte

Durch dieses Feature wird das personenbezogene Datum cardholder, also der Karteninhaber der Kreditkarte, dem Merchant übermittelt.

2. Rückgabe Maskierte Kartennummer Kreditkarte

Durch dieses Feature wird das personenbezogene Datum maskedPan, also 10 der insgesamt 16 bis 19 Stellen der Kreditkartennummer, dem Merchant übermittelt.

3. Rückgabe Referenznummer der Transaktion

Durch dieses Feature erhält der Merchant über den Parameter gatewayReferenceNumber eine eindeutige Referenznummer der Transaktion. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.

4. Rückgabe iDEAL Konsumentendaten

Durch dieses Feature werden dem Merchant nähere Infos zum Endkunden übermittelt, die alle personenbezogene Daten sind: idealConsumerData.

5. Rückgabe PayPal Konsumentendaten

Durch dieses Feature werden dem Merchant nähere Infos zum Endkunden übermittelt, die alle personenbezogene Daten sind: paypalConsumerData.

6. Rückgabe Vertragsnummer, auf welche die Transaktion gelaufen ist

Dieses Feature übermittelt über den Parameter gatewayContractNumber die Vertragsnummer des Prozessors / Acquirers des Merchants nach erfolgreicher Zahlung an sein System. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.

7. Rückgabe Sofort Konsumentendaten

Durch dieses Feature werden dem Merchant nähere Infos zum Endkunden übermittelt, die alle personenbezogene Daten sind: sofortConsumerData.

8. Rückgabe SEPA Konsumentendaten

Durch dieses Feature werden dem Merchant nähere Infos zum Endkunden übermittelt, die alle personenbezogene Daten sind: SepaConsumerData.

9. Rückgabe Ablaufdatum der Kreditkarte

Dieses Feature liefert dem Merchant das Ablaufdatum der verwendeten Kreditkarte über den Parameter expiry. Dieses Datum lässt allerdings isoliert keine Rückschlüsse auf Personen zu.

10. Selektive automatische Buchung

Über den Parameter #autodeposit kann der Merchant für jede Transaktion einzeln bestimmen, wann die Zahlung durchgeführt wird. Dies ist eine Konfigurationseinstellung über einen Schritt, der im Rahmen der Zahlungsabwicklung ohnehin zu erfolgen hat. Der Parameter legt nur den Zeitpunkt dieses Schrittes fest. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.

11. Auswahl der Kartenmarke

Mit diesem Feature kann der Merchant über den zusätzlich zu übermittelnden Parameter financialInstitution entscheiden, welche Kartentypen zur Zahlung zugelassen werden. Dadurch kommt es jedoch zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.

12. Eigene Auftragsnummer

Zu Zwecke organisatorischer Erleichterung kann der Merchant über den Parameter orderNumber jede Transaktion eindeutig zuordnen. Rückschlüsse auf Personen sind dadurch jedoch nicht möglich.

Zusatzfeatures

Für jedes zusätzlich zum Basisprodukt Checkout Seamless bestellte Feature ist hier ersichtlich, welche Daten das jeweilige Feature verarbeitet.

Sämtliche für das jeweilige Feature angegebenen Parameter müssen verarbeitet werden, um dem Merchant zusätzlich zum Basisprodukt das gewünschte Feature zur Verfügung stellen zu können (Zweck der Datenverarbeitung).

Table 29. Optimierung der Usability
Feature Verarbeitete Daten und Datenschutzinfos

1. Individuelles Layout für Checkout Seamless

Das Feature ermöglicht dem Merchant die Gestaltung von Checkout Seamless im Corporate Design. Die Layoutanpassung erfolgt über vorgefertigtes CSS-Template, das der shopId des Merchants hinterlegt wird. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.

2. Zusätzliche Sprachen

Um für jede Transaktion das Bezahlfenster in der für den Endkunden präferierten Sprache anzuzeigen, werden die Spracheinstellungen vom Merchant über den Browser des Endkunden erhoben und hobex Service IT durch den Merchant über den Parameter language mitgeteilt. Als personenbezogenes Datum wird somit die Sprache des Endkunden verarbeitet, auch wenn der Endkunde mangels Kenntnis von Personendaten unbekannt bleibt.

3. Zusätzliche Währungen

Abhängig vom Produktkatalog des Merchants sind bestimmte Produkte in 1 bis N Währungen zahlbar. Je nach Produkt und Auswahl des Endkunden wird die abzuwickelnde Währung hobex Service IT über den Request Parameter currency mitgeteilt. Als personenbezogenes Datum wird somit die Währung der abgewickelten Zahlungstransaktion des Merchants verarbeitet.
Table 30. Optimierung interner Prozesse
Feature Verarbeitete Daten und Datenschutzinfos

4. Wiederkehrende Zahlung

Mit diesem Feature kann der Merchant über die Backend-Operation recurPayment wiederkehrende Zahlungen durchführen. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.

5. Weitere Zugänge zum Payment Center

Für die Web-Applikationen des Händlerlogins wird ein neuer Zugang für den Merchant angelegt, der nicht personengebunden ist. Für den Zugang ist ein Benutzername und ein Passwort erforderlich. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten.)

6. Rückgabe der Kreditkartennummer als Hash-Wert

Durch dieses Feature wird dem Merchant der Zufallswert hashedPan übermittelt, der den Merchant erkennen lässt, ob für unterschiedliche Zahlungstransaktionen dieselbe Kreditkartennummer verwendet wurde. Dieses Datum lässt allerdings keine Rückschlüsse auf eine Person zu (z.B. wem gehört die Kreditkartennummer?).

7. Erweiterung für Checkout Seamless zur Backoffice Nutzung

Durch dieses Feature wird der Merchant in die Lage versetzt, das Produkt wie ein Endkunde zu nutzen. Eine weitere Verarbeitung von Datenkategorien ergibt sich dadurch jedoch nicht.
Table 31. Sicherheitsfeatures
Feature Verarbeitete Daten und Datenschutzinfos

8. Zustellung der Zahlungsbenachrichtigung per E-Mail

Durch dieses Feature erhält der Merchant sämtliche Response Parameter, die er ansonsten über die confirmUrl erhält, im Email-Format sowie an die gewünschte Email-Adresse. Dadurch kommt es zu keiner zusätzlichen Verarbeitung von personenbezogenen Daten. Die entsprechenden Response Informationen sind hier ersichtlich.

9. Wiederholte Zustellversuche der Zahlungsbenachrichtigung

Für den Fall, dass der Server des Merchants nicht erreichbar sein sollte, ermöglicht dieses Feature die erneute Zustellung der Zahlungsbenachrichtigung. Diese Konfigurationseinstellung bewirkt jedoch keine zusätzliche Verarbeitung von personenbezogenen Daten.
