Information on the protection of personal data for Checkout Terminal

Information on the protection of personal data for Checkout Terminal

This chapter provides information on all data that are processed by our solution Checkout Terminal.

Checkout Terminal data processing volume increases with the services contracted. When ordering Checkout Terminal the merchant may acquire:

  • 1 basic solution, optionally extendable with

  • 1 to N payment methods, and/or

  • 0 to N optional and additional features

Additionally, the general functioning of the product Checkout Terminal-both in its basic version and when using different payment methods-is also described in detail here: Using Checkout Terminal

Data entered by the merchant

Additional Features

This chapter provides information on the data processed for each feature that has been added to the basic Checkout Terminal solution.

Feature Processing of data and information on the protection of personal data

1. Additional access to the Payment Center

For the web application of the link:https://hobex Service IT.at/haendler-login/[Merchant Login,window=\"_blank\"] a new access is set up which does not involve any personal data. A username and password are required for the access. Thus, no additional processing of personal data takes place.

2. Additional currencies

Depending on the merchant’s product portfolio, certain products may be paid in 1 to N currencies, which are defined by hobex Service IT in the configuration setting. Thus, no additional processing of personal data takes place. However, the currency of the payment itself is considered personal data that relates to the currency of the processed transaction of the merchant.

3. Recurring payment

Based on a configuration setting provided by hobex Service IT, this feature allows the merchant to carry out recurring payments. A recurring payment is carried out by selecting option "Recurring Payment" and stating the respective order number as described in detail in the Checkout Terminal Documentation. Thus, no additional processing of personal data takes place.

4. Immediate deposit

This configuration setting allows an immediate deposit (instead of an authorization of the transaction with subsequent manual deposit). Thus, no additional processing of personal data takes place.
