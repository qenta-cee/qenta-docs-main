Demo Mode and Demo Data

Demo Mode and Demo Data

To test the integration of the QPAY Checkout Page within your web shop we can offer you a demo mode with some demo data to check if your integration fulfills the requirements of the interfaces.

During demo mode there will be no communication to the financial providers and the payment process is only simulated and no money is transferred. Additionally your executed payments are not displayed within the QPC Payment Center.

Configuration of Demo Mode in your Web Shop

To use the QPAY Checkout Page in demo mode during the integration to your web shop, please use the following values for the parameter customerId and the secret for generating the fingerprint:

QPAY Checkout Page - Demo Mode

customerId

D200001

secret

B8AKTPWBRMNBV455FG6M2DANE99WU2

Please do not send the secret as a request parameter from your web shop to the QPAY Checkout Page!

For changing from demo mode to production mode you will only need to update the request parameter customerId and your secret for generating the fingerprint with the values you will get from our Support team after closing the contract with QENTA.

Before integration you may want to have a look at our Demo Shop: https://www.wirecard.at/en/products/qpay/qpay-demoshop/ where you are also able to use the demo data described on this page.
