Change log

for xt:Commerce 4/5 and Wirecard Checkout Page

Version 1.6.0

Version 1.6.7 (2018-09-06)

  • Compatibility with XT:Commerce 5.1.3

Version 1.6.6 (2018-05-15)

  • Update amount setter with thousand separator.

  • Force sending shipping/billing data for invoice/installment.

  • Update pre-checks for invoice/installment.

Tested versions: xt:Commerce 5.0.08 with PHP 5.6, PHP 7.0 and PHP 7.0.27

Version 1.6.5 (2018-03-15)

  • Calculate device fingerprint only when Ratepay is enabled for invoice and installment.

Tested versions: xt:Commerce 5.0.07 with PHP 5.6, PHP 7.0 and PHP 7.0.27

Version 1.6.4 (2017-12-20)

  • Remove payment method @Quick.

  • Prevent issue with payment cancelation.

  • Prevent issue with Sofort. redirect.

Version 1.6.3 (2017-10-16)

  • Prevent possible bug in combination with other payment methods.

Version 1.6.2 (2017-10-04)

  • Rename payment method SOFORT Banking to Sofort./Online bank transfer.

  • Change "SOFORT Banking" logo to Klarna logo.

Version 1.6.1 (2017-08-29)

  • Add new parameter consumerDeviceId.

Version 1.6.0 (2017-07-31)

  • Add client library.

  • Remove deprecated payment methods (mpass, Skrill Direct).

  • Update payment method logos.

  • Update payment method wordings.

  • Add pre-checks for payment methods Invoice and Installment as well as input fields on front-end.

  • Add basket parameter.

  • Add checkbox for payolution terms.

  • Send selected financial institutions for payment methods iDEAL and eps-Überweisung together with other init parameters.

Version 1.5.2 (2017-02-27)

  • Add compatibility with PHP 7.0

Tested versions: xt:Commerce 5.0.05 with PHP 5.6 and PHP 7.0

Version 1.5.1 (2016-11-28)

  • Remove additional form fields in checkout template.

  • Update parameter orderNumber in config.

  • Add check for xt:Commerce 4 compatibility.

Tested version: xt:Commerce 4 and xt:Commerce 5.0 with PHP 5.6

xt:Commerce 5.0.05 with PHP 5.6 and PHP 7.0

Version 1.5.0

  • Initial version of plugin v1.5.0 based on xt:Commerce 4 including change log (dated 2016-10-24).

  • Add xt:Commerce5 compatibility.

  • Add images for paymentTypes.

  • Add xt_responsive pages.

  • Implement relevant hookpoints.

  • Improve security on payment server response.

  • Add parameter consumerMerchantCrmId.

Tested version: xt:Commerce 5.0 with PHP 5.6
