SOFORT Banking

SOFORT Banking

General information

To integrate and offer the payment type SOFORTUEBERWEISUNG for your consumers in your web shop you need to fulfill some important requirements:

  • Register and setup a project for Wirecard CEE to your SOFORT account as described below.

  • Use the specific response parameters as defined below within the checkout process of your web shop.

If these requirements are not fulfilled, please do not offer this payment type to your consumer within your web shop.

You can find more details regarding SOFORT at https://www.sofort.com.

Setup of 3rd party permissions to your merchant account and project

To enable the integration of SOFORT to your web shop within the Wirecard Checkout Page you need to register and set up a new project.

This can be done within the following steps:

  1. Register yourself at www.sofort.com as a new merchant (if you have not done this already).

  2. Create a new "SOFORT-Classic-Project" (also named "SOFORT Überweisung Project").

  3. Set "Wirecard CEE (Payment Service Provider)" as Shop-System.

  4. Set "-USER_VARIABLE_0-" as value for the success link.

  5. Set "automated redirection" to active.

  6. Set "-USER_VARIABLE_1-" as value for the cancel link.

  7. Save your newly created project.

  8. Within the additional configuration of this project set the input paramenter amout to non-changeable.

  9. Create a new notification via HTTP with URL "-USER_VARIABLE_2-" and method POST.

  10. Activate this notification and save it.

  11. Please set a password for the project and for notifications and select a hash algorithm.

  12. Finally, please communicate (either by phone or fax) the following data to us:

    • your customer id

    • your project number

    • your project password

    • your notification password

    • and your selected hash algorithm

Additional response parameters

Within the usage of this payment type in your web shop the Wirecard Checkout Page will return additional parameters to your web shop. These parameters are specific to this payment type and are only returned when your consumer selects it in your web shop.

Parameter State Data type Short description

securityCriteria

success

numeric with a fixed length of 1

security criterion returned by sofortüberweisung where “1” means the criteria were met and “0” means criteria were not met, in that case you should wait until the amount has been received on your side before sending goods or grant access to digital media

senderAccountNumber

success

numeric with a variable length of up to 30

account number of sender as returned by sofortüberweisung

senderAccountOwner

success

alphanumeric with special characters and a variable length of up to 255

owner of account as returned by sofortüberweisung

senderBankName

success

alphanumeric with special characters and a variable length of up to 255

name of bank as returned by sofortüberweisung

senderBankNumber

success

numeric with a variable length of up to 30

number of bank as returned by sofortüberweisung

senderBIC

success

alphanumeric with a variable length of up to 50

BIC of sender as returned by sofortüberweisung

senderCountry

success

alphabetic with a fixed length of 2

country code of sender as returned by sofortüberweisung

senderIBAN

success

alphanumeric with a variable length of up to 50

IBAN of sender as returned by sofortüberweisung

FAQ

What is displayed as reason for payment within the account statement?

The reason for payment can be displayed either within a two-line statement or a one-line statement:

Within the two-line statement in the first line will be the value of optional request parameter orderReference and in the second line will be the value of optional request parameter customerStatement.

[orderReference]
[customerStatement]

The one-line statement would look like:

[orderReference] [customerStatement]

If one or both of these optional request parameters are not set, the following content will be displayed instead:

  • Value of the response parameter orderNumber instead of orderReference.

  • The text ''Order '' and the value of the response parameter orderNumber instead of customerStatement.

  • If orderReference is not set or empty, the value of the response parameter orderNumber will be used instead.

  • If customerStatement is not set or empty, the following text will be displayed: Order [orderNumber]

For example if you set orderReference to "abc123" and do not set customerStatement the result will be:

abc123 Order 12345678

or in the two-line version:

abc123
Order 12345678

Please be aware that within SEPA each line of text may not exceed 35 characters.
