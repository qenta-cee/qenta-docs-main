Information on the protection of personal data for QCALL Checkout Terminal

Information on the protection of personal data for Qcall Checkout Terminal

This chapter provides information on all data that are processed by our solution QCALL Checkout Termianl (QCALL).

QCALL Checkout Terminal data processing volume increases with the services contracted. When ordering QCALL Checkout Terminal, you may acquire

  • 1 basic solution, optionally extendable with

  • 1 to N payment methods, and/or

  • 0 to N optional and additional features

Go to section Data entered by the merchant for an overview on the processing of data (parameters) for a given payment transaction and the selected payment method. Furthermore, section Additional features provides additional information on the processing of data (parameters) for each feature that was added to the basic solution.

Additionally, the general functioning of the product QCALL - both in its basic version and when using different payment methods - is also described in detail here: Using QCALL

Additional features

This chapter provides information on the data processed for each feature that has been added to the basic QCALL Checkout Automated solution.

Feature Processing of data and information on the protection of personal data

1. Additional access to the QPC Payment Center

For the web application of the Merchant Login a new access is set up which does not involve any personal data. A username and password are required for the access. Thus, no additional processing of personal data takes place.

2. Additional currencies

Depending on the merchant’s product portfolio, certain products may be paid in 1 to N currencies, which are defined by QENTA in the configuration setting. Thus, no additional processing of personal data takes place. However, the currency of the payment itself is considered personal data that relates to the currency of the processed transaction of the merchant.

3. Recurring payment

Based on a configuration setting provided by QENTA, this feature allows the merchant to carry out recurring payments. A recurring payment is carried out by selecting option "Recurring Payment" and stating the respective order number as described in detail in the QCALL Documentation. Thus, no additional processing of personal data takes place.

4. Immediate deposit

This configuration setting allows an immediate deposit (instead of an authorization of the transaction with subsequent manual deposit). Thus, no additional processing of personal data takes place.
