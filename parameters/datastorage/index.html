Data Storage Parameters

Data Storage Parameters

Request Parameters

To initialize the hobex Data Storage the merchant has to set all required parameters to their corresponding values that are needed within the online shop. If one or more of these required parameters are missing an error message will be accrued.

Table 1. Initialization of hobex Data Storage
Parameter Data type Description

customerId

Alphanumeric, fixed 7

Unique ID of merchant.

orderIdent

Alphanumeric

Unique reference to the order of the consumer.

returnUrl

Alphanumeric

Return URL for outdated browsers.

language

Alphabetic, fixed 2.

Language for returned texts and error messages.

requestFingerprint

Alphanumeric, 128.

Computed fingerprint of the parameter values and the secret.

Table 5. Initialization of hobex Data Storage GooglePay
Parameter Data type Short description

googlepaymerchantid

Alphanumeric, 16

Unique ID of hobex. Will be provided by hobex.

googlepaymerchantname

Alphanumeric

Unique Name for the Merchant. Will be provided by hobex.

googlepayallowedauthmethod

Enumeration

Value must be forwareded as = "PAN_ONLY", "CRYPTOGRAM_3DS"

googlepayallowedcardnetwork

Enumeration

Supported Brands for GooglePay. Eg.: If AMEX, VISA and Mastercard should be accepted use as value = AMEX","MASTERCARD","VISA". Brands and GooglePay needs to be supported by the credit card acquirer. Available Brands will be provided by hobex.

googlepaycurrencycode

Alphabetic

Currency

googlepaycountrycode

Alphabetic

ISO 3166-1. Will be provided by hobex.

googlepaytotalprice

Numeric

Total amount, only a dot(.) allowed as seperator.

javascriptScriptVersion

Alphanumeric, fixed 9 characters

The value googlepay must be set to allow the rendering of googlePay.
