paysafecard

paysafecard has announced to introduce a new parameter - consumerMerchantCrmId. In order to ensure more transparency and security in the payment process, this new parameter is mandatory for all payments as of February 6th, 2017. See paysafecard#paysafecard_announcement below.

The parameter is understood as unique ID identifying the consumer in your online shop, e.g. using data from your Customer Relationship Management (CRM) system. However, paysafecard must not receive any data or information that allow to draw any conclusions on the final consumer.

The parameter must not be a human readable e-mail address. However, if an e-mail address is used it has to be hashed beforehand by the merchant to comply with the directives on information security and data protection.

As of February 6th, 2017 payments can only be processed if this parameter is used.

Please note that paysafecard refers to the new parameter as merchantClientID. In QENTA interfaces and documentation consumerMerchantCrmId is used.

Technical information about the implementation of consumerMerchantCrmId can be found below. For any additional information on the introduction of this new parameter please contact paysafecard.

paysafecard and WCP or WCS

The parameter consumerMerchantCrmId has to be set by the merchant using QPAY Checkout Page or QMORE Checkout Seamless for initiations via QPAY, QPAY server-to-server initiations and QMORE front-end initiation.

Also, the parameter consumerMerchantCrmId is part of the fingerprint.

The following table shows an overview of the required request parameter regarding paysafecard.

Parameter Data type Short description

consumerMerchantCrmId

Alphanumeric with a variable length of up to 50 characters.

Unique ID identifying the consumer of your online shop, e.g. from your CRM system. The parameter must not be a human readable e-mail address. However, if an e-mail address is used it has to be hashed beforehand by the merchant.

The following table shows an overview of all optional request parameters regarding paysafecard.

Parameter Data type Short description

customerStatement

ASCII with a variable length of up to 254 characters.

Text displayed on bank statement issued to your consumer by the financial service provider.

orderReference

ASCII with a variable length of up to 128 characters.

Unique order reference ID sent from merchant to financial institution.

Refer also to our parameter specific information in chapter paysafecard.

paysafecard and QFILE/Qtill

The following information is intended ONLY for merchants using payment method paysafecard and QENTA Checkout Enterprise.

In order to set the new parameter consumerMerchantCrmId, merchants using QTILL Checkout Enterprise must use the most recent JAPI version. Please update your JAPI version accordingly.

Click JAPI to download the latest JAPI version.

For any additional information or help contact our support teams.

paysafecard announcement

Our records indicate that you have unfortunately submitted the incorrect merchantClientID/customerID for your transactions.

Please ensure that the merchantClientID is always 1:1 in relation to your customers. That means: Every customer has the same merchantClientID for each of their transactions in your online shop, but two clients never use the same merchantClientID.

Important: In addition, it must be ensured that the merchantClientID contains no personal information of any kind (e.g. user name, email address, etc.). Should you wish to use such information, you must ensure that it always remains encrypted - ideally using, for example, an internal customer reference number.

Given the above information, we are hereby notifying you that from early 2017, for regulatory reasons, paysafecard will be refusing any payment that does not contain a valid merchantClientID, as this will be absolutely mandatory.

Please put your resources into motion so as to ensure the necessary adjustments can be implemented as soon as possible.

Should you have any questions regarding this, please do not hesitate to contact our integration team (techsupport@paysafecard.com). We are happy to be of further assistance!
