URL Migration to hobex.at

URL Migration to hobex.at: Technical Update for Your Integration

As part of the technical integration of QENTA into hobex, central product and service URLs will be migrated. This change affects selected products and is mandatory. The previous *.qenta.com domains will be replaced by new *.hobex.at domains.

✅ 1. No Action Required for the web based Products

The following products/services will be automatically redirected. You can continue to use your existing links—no changes required:

Name old URL new URL

Checkout Terminal

https://portal.qenta.com/terminal

https://portal.hobex.at/terminal

Checkout Automated

https://portal.qenta.com/automated

https://portal.hobex.at/automated

PaybyLink

https://portal.qenta.com/portal

https://portal.hobex.at/portal

Payment Center

https://portal.qenta.com/paymentcenter

https://portal.hobex.at/paymentcenter

Checkout Journal

https://portal.qenta.com/journal

https://portal.hobex.at/journal
You may update your bookmarks voluntarily, but no technical action is needed.

⚡️ 2. Manual Domain Change Required for the Following Products

For the following integrations, the domain must be updated by September 1, 2025. As of this date, all requests will no longer be redirected to api.qenta.com but answered directly from papi.hobex.at. The new domains will be available starting June 9, 2025, and may be tested using existing test CustomerIDs.

In all examples below, only the domain changes from api.qenta.com to papi.hobex.at. The rest of the path remains the same.
The IP range does not change and remains the same: Firewall Settings

🔒 3. Certificate Update for Checkout Enterprise

In addition to the domain change, Checkout Enterprise requires a new truststore certificate:

Truststore Certificate: Download. Please ensure the new certificate is configured accordingly.

papi.hobex.at certificate (if required): Download.

Endpoints

  • Within qtill.properties values have to changed within QTILL_HOST_URL.

  • Settings accept only one URL.

Old Endpoints New Endpoint Notes

https://api.qenta.com/enterprise

https://papi.hobex.at/enterprise

https://api.qenta.com/enterprise-demo

https://papi.hobex.at/enterprise-demo
Be aware to use standard https port 443 and keep firewall settings in mind during updates.
The IP range does not change and remains the same: Firewall Settings

Examples:

Old configuration
TRUSTSTOREFILE=/usr/local/qenta/qtill/qtill-trust-old.jks
TRUSTSTOREPASSWORD=<your productive truststore password>
New configuration
TRUSTSTOREFILE=/usr/local/qenta/qtill/qtill-trust.jks
TRUSTSTOREPASSWORD=<your productive truststore password>
Additional changes in settings:
settings.setTruststoreFile(new File("qtill-trust.jks"));
settings.setTruststorePassword("<truststore password>");


Old configuration

QTILL_HOST_URL=https://api.qenta.com/enterprise-demo

New configuration

QTILL_HOST_URL=https://papi.hobex.at/enterprise-demo

Addtional Changes in QTillSettings:

QTillSettings settings = QTillSettings(new URL("https://papi.hobex.at/enterprise-demo"));
